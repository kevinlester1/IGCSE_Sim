1. Types of Program Errors

Syntax Errors

These errors occur when the code violates the rules of the programming language. The program won't run at all.

OUTPUT "Max number: , MaxNum
Explanation: The comma is inside the string literal. The compiler doesn't understand this instruction and cannot run the program. The fix is: OUTPUT "Max number: ", MaxNum.

Logical Errors

The program runs, but produces an incorrect result. Find the bug in this algorithm that's supposed to find the largest of three numbers.

INPUT Num1, Num2, Num3
MaxNum ← 0
IF Num1 >= Num2 AND Num1 >= Num3 THEN
    MaxNum ← Num3
ELSE IF Num2 >= Num3 THEN
...

Try testing with Num1 as the largest value.

Runtime Errors

The program is syntactically correct, but a specific input causes it to crash while running. Can you make this program crash?

INPUT NumberOfCakes
INPUT NumberOfPeople
CakesPerPerson ← NumberOfCakes / NumberOfPeople
OUTPUT CakesPerPerson

Enter the number of cakes and people.

2. Types of Test Data

Consider a program that accepts a whole number between 10 and 50 (inclusive). We use different types of data to test it robustly.

Normal Data

Valid, typical data to check for correct outputs.

Examples: 20, 35, 49

Extreme / Boundary Data

Data at the very limits of the valid range (and just outside) to catch comparison errors.

Valid: 10, 50 | Invalid: 9, 51

Abnormal Data

Data of the wrong type or format that the program should handle gracefully.

Examples: "hello", -100, 3.14

Interactive Trace Tables

Example 1: A `WHILE` Loop

Trace this algorithm that uses a `WHILE` loop to see how variables change.

Algorithm:


Num ← 3
N ← 0
WHILE N < 4 DO
  Num ← Num + N
  N ← N + 1
END WHILE
OUTPUT Num

Example 2: A `FOR` Loop

This example uses a `FOR` loop. Watch how `Count` increments automatically.

Algorithm:


Total ← 0
FOR Count ← 1 TO 3
  Total ← Total + (Count * 2)
END FOR
OUTPUT Total

Example 3: `IF/ELSEIF/ELSE` Logic

This trace shows how conditional statements work. The program will only execute the code inside the first true condition.

Algorithm:


Score ← 75
Grade ← ""
IF Score > 80 THEN
  Grade ← "Distinction"
ELSEIF Score > 65 THEN
  Grade ← "Merit"
ELSE
  Grade ← "Pass"
END IF

Example 5: Array Processing

This algorithm finds the largest number in a list (array). Watch how `MaxValue` is updated only when a larger number is found.

Algorithm:


Numbers ← [12, 5, 23, 9]
MaxValue ← Numbers[0]
FOR i ← 1 TO 3
  IF Numbers[i] > MaxValue THEN
    MaxValue ← Numbers[i]
  END IF
END FOR
OUTPUT MaxValue
