A2.2.3 Networking Models

Compare how data and processing are distributed. Real-world applications vary based on the model's benefits and drawbacks.

Client-Server

Centralized control, easier to secure and backup (e.g., Banking, Web Browsing). Prone to bottlenecks.

Peer-to-Peer (P2P)

Decentralized, highly scalable, no single point of failure (e.g., File Sharing, Blockchain, VoIP). Harder to secure.
