IB A4.3.8 Syllabus Guide

Structure and function of ANNs and how multi-layer networks model complex patterns.

1. Structure and Function of ANNs

An Artificial Neural Network is a computational model inspired by the biological brain. It simulates interconnected nodes (neurons) that process data, learn from it, and make predictions.

In this simulation, the ANN’s function is classification — deciding whether to "Hire" (1) or "Reject" (0) a candidate. By adjusting how much nodes care about certain traits, the network "learns" the complex pattern of what makes a successful employee. ANNs enable tasks such as:

  • Classification: Grouping data into discrete categories (e.g., Hire vs. Reject, Spam vs. Not Spam).
  • Regression: Predicting a continuous numerical value (e.g., predicting the exact salary a candidate should receive).
  • Pattern Recognition: Identifying trends or structures in complex, unstructured data (e.g., recognizing that candidates with high tech and high soft skills usually succeed).

2. The Single Perceptron

A single perceptron is the fundamental building block of an ANN. You can understand this using just the Tech Lead node from the simulator. It consists of five distinct parts:

  1. Inputs ($x_i$): The raw data feeding into the neuron. In the simulation, this is the candidate's resume: Tech Skills ($x_1$), Soft Skills ($x_2$), and Experience ($x_3$).
  2. Weights ($w_i$): The "importance" assigned to each input. For example, the Tech Lead places a high positive weight (e.g., $1.5$) on Tech Skills, but $0.0$ on Soft Skills.
  3. Bias ($b$): The neuron's inherent threshold or "strictness." A negative bias (like $-1.5$) means the candidate starts at a deficit and must heavily impress the interviewer to get a positive rating.
  4. Summation & Activation Function: The perceptron calculates the weighted sum of all inputs and adds the bias:
    $$Sum = (x_1 \cdot w_1) + (x_2 \cdot w_2) + (x_3 \cdot w_3) + b$$
    This sum is then passed through an Activation Function (the "Decision Rule"), like ReLU. If the sum is positive, the Tech Lead gives a thumbs up; if negative or zero, they remain silent.
  5. Output: The final signal passed forward (e.g., "Impressed" or "Unimpressed").

3. The Multi-Layer Perceptron (MLP)

A single perceptron is limited—it can only solve linearly separable problems. What if a role requires a candidate to be either a pure technical genius or an experienced manager with great soft skills, but not a mediocre mix of both? A single node cannot model this logic (known mathematically as the XOR problem).

This is why we need a Multi-Layer Perceptron (MLP). You can outline this using the three distinct sections of the simulator:

1. Input Layer

The Resume: This layer does no math; it simply holds the initial state of the environment (the candidate's skills) and passes it forward to the network.

2. Hidden Layer(s)

The Committee: Instead of looking at raw data, hidden layers extract abstract features. Because the Tech Lead and Culture Lead look for different traits simultaneously, they can capture complex, overlapping patterns.

3. Output Layer

The Boss: Makes the final decision. The Boss does not look at the raw resume; they only evaluate the combined abstract features identified by the hidden layer.

Weights ($w$)

The "Importance"

Determines how much influence an input has on the output.

Hiring Analogy: "How much does this Interviewer care about this skill?"

Bias ($b$)

The "Threshold"

Shifts the activation curve. A negative bias acts as a strict threshold.

Hiring Analogy: "Is the interviewer strict (negative bias) or easygoing (positive bias)?"

Hidden Layer

The "Features"

Layers between input and output. They transform raw data into abstract features.

Hiring Analogy: The "Committee Members". Each looks for different traits (Tech vs Culture).

Activation Function

The "Decision Rule"

Mathematical formula deciding if a neuron "fires".

Hiring Analogy: How they express opinion.
ReLU: "Silent unless impressed."
Sigmoid: "I'm 80% sure."

Forward Propagation

The "Evaluation"

Passing data from inputs to output to get a prediction.

Hiring Analogy: Reading resume -> Forming opinions -> Making final hiring decision.

Backpropagation

The "Learning"

Calculating error and adjusting weights backwards to improve.

Hiring Analogy: The feedback meeting. "We made a bad hire (Error). Tech Lead, you were too lenient (Adjust Weight)."
