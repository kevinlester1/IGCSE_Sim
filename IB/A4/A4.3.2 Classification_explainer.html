What is Classification?

Unlike regression (which predicts a continuous number, like price), classification is a supervised learning technique used to predict discrete categorical outcomes. The goal is to assign new data points to one of several predefined groups or "classes" based on patterns learned from labeled training data.

Regression

Predicts a quantity on a continuous scale.

  • Temperature tomorrow ($24.5^\circ C$)
  • Price of a house ($\$450,000$)
  • Age of a person ($34$ years)

Classification

Predicts a category (a discrete label).

  • Email: Spam or Not Spam
  • Transaction: Fraudulent or Legitimate
  • Animal Image: Cat, Dog, or Bird
The core challenge: Given historical data where we already know the correct labels (categories), how do we build an algorithm that can accurately guess the label for a brand new, unseen piece of data?

K-Nearest Neighbours (K-NN)

K-NN is one of the simplest and most intuitive classification algorithms. It operates on a simple premise: "Birds of a feather flock together." Data points with similar characteristics tend to belong to the same category.

How the Algorithm Works

1

Measure Distance

When a new, unlabelled data point arrives, the algorithm calculates the mathematical distance (e.g., straight-line Euclidean distance) between this new point and every other point in the training dataset.

2

Find the 'K' Nearest

It selects the K closest data points. K is a number chosen by the user (e.g., 3, 5, or 7). For binary classification, K is usually an odd number to prevent ties.

3

Majority Vote

The algorithm looks at the known classes of those K neighbours. The new point is assigned to the class that is most frequent among them (a simple majority vote).

Note: K-NN is often called a "lazy learner" because it doesn't actually build a complex model during a training phase. It simply stores the dataset and does all the heavy mathematical lifting only when it needs to make a prediction.

Decision Trees

A Decision Tree algorithm classifies data by asking a series of hierarchical, yes/no questions based on the features of the data. It maps out paths to different conclusions, resembling an upside-down tree.

Root Node

The very first question. The algorithm determines the best feature to split the data initially to create the most distinct groups.

Decision Nodes

Subsequent questions where branches split. Each node evaluates a specific feature of the data point.

Leaf Nodes

The end of a branch. These represent the final predicted class or category (e.g., Apple, Orange).

Real-World Applications

Classification algorithms are the engines behind many systems we use daily. Here is how K-NN and Decision Trees are applied in practice.

Collaborative Filtering

Powered by K-NN

Streaming services (like Netflix or Spotify) and e-commerce platforms use K-NN for recommendation systems. If you want to predict if User A will like a movie (a discrete outcome: Like / Dislike), the system finds the K users who have historically rated movies most similarly to User A.

The Logic

"Find the 5 users whose tastes are the closest distance to mine. If 4 out of those 5 users liked 'Sci-Fi Movie X', classify it as a 'Like' for me and recommend it."

Medical Diagnosis

Powered by Decision Trees

In healthcare, Decision Trees are highly valued because they are interpretable. A doctor can clearly trace the logical path the model took to reach a diagnosis (a discrete outcome: Disease A, Disease B, or Healthy) based on a patient’s symptoms.

The Logic

Root node: "Temperature > $38^\circ C$?" -> Yes -> "Cough present?" -> Yes -> "Fatigue level high?" -> Yes -> Leaf Node Prediction: Flu.

Moving Beyond 2D: High-Dimensional Spaces

In our earlier examples, we visualized data with only two features (X and Y axes). However, real-world classification datasets often have dozens, hundreds, or even thousands of features (e.g., classifying a medical image based on the thousands of pixels it contains). We call each feature a dimension.

How Distance Scales

The beauty of K-Nearest Neighbours is that the underlying math works exactly the same way regardless of how many dimensions you have. The Euclidean distance formula simply expands to include the extra features:

d = √((x2-x1)² + (y2-y1)² + (z2-z1)² + ... + (n2-n1)²)

Visualizing 3 Dimensions

Once we move past 3 dimensions, humans can no longer visually map the space. But mathematically, the algorithm continues to find the "closest" points in a multi-dimensional hyper-space seamlessly.
