    Algorithm Flowchart

    This flowchart shows the main loop of the genetic algorithm. Click on any stage to see a detailed illustration of that concept.

    Illustration: Initialization

    Initialization is the very first step. A population of potential solutions (chromosomes) is created randomly to provide the initial gene pool for the algorithm to work with.

    Creating a Random Population

    Given a set of cities, the algorithm creates a large number of completely random routes. The size of this population is set by the "Population Size" parameter in the simulation.

    • Route 1: [ London, Paris, Rome, Berlin, Madrid, ... ]
    • Route 2: [ Madrid, Rome, London, Berlin, Paris, ... ]
    • Route 3: [ Berlin, London, Madrid, Paris, Rome, ... ]
    • ... and 497 more random routes ...

    This random collection of routes, most of which are very inefficient, serves as Generation 0. From here, the process of selection and evolution begins.

    Illustration: Fitness Function

    The fitness function evaluates how "good" each solution is. For the Traveling Salesperson, a shorter total distance equals a higher fitness.

    Route A (Less Fit)

    Total Distance: ~258

    Route B (More Fit)

    Total Distance: 240

    The genetic algorithm aims to maximize fitness. Since a shorter route is better, fitness is calculated as 1 / Total Distance. Therefore, Route B has a higher fitness: 1 / 240 ≈ 0.0041 vs. 1 / 258 ≈ 0.0038 for Route A.

    Illustration: Selection

    Selection is the process of choosing which routes from the current generation will become parents for the next. This is based on the principle of "survival of the fittest"—routes with shorter distances (higher fitness) are more likely to be chosen.

    1. Population Ranked by Fitness

    After calculating the distance for every route in the population, they are ranked from best to worst.

    • Route 1 (Distance: 1250) → Elite
    • Route 2 (Distance: 1280) → Selected for Mating Pool
    • Route 3 (Distance: 1310) → Selected for Mating Pool
    • Route 4 (Distance: 1450)
    • Route 5 (Distance: 1520)
    • ... many more routes ...

    2. Creating the Next Generation

    Elitism: The very best routes are copied directly. See the Elitism tab for details.

    Mating Pool: Other high-fitness routes are selected to be in the "mating pool." Parents will be chosen randomly from this pool to create new child routes via Crossover.

    Illustration: Elitism

    Elitism is a crucial part of the selection process. It guarantees that the best solution(s) from the current generation are not lost and are carried over to the next generation exactly as they are.

    Current Generation (Ranked)

    • Route 1 (Distance: 1250)
    • Route 2 (Distance: 1280)
    • Route 3 (Distance: 1310)
    • Route 4 (Distance: 1450)
    • ...

    Next Generation (Starting Population)

    With an Elitism Rate of 10% on a population of 20, the top 2 routes are preserved.

    • Route 1 (Distance: 1250)
    • Route 2 (Distance: 1280)
    • ... (The other 18 slots will be filled by new children from crossover and mutation)

    Illustration: Crossover

    Crossover combines two parent routes to create a new child route, mixing their genetic information to potentially create an even better route.

    Parent 1

    [ London, Paris, Rome, Berlin, Madrid, Stockholm ]

    Parent 2

    [ Madrid, Stockholm, London, Paris, Rome, Berlin ]

    ↓

    Creating the Child

    1. A random segment is taken from Parent 1. In this case, we'll take Paris, Rome, Berlin.

    [ _, Paris, Rome, Berlin, _, _ ]

    2. The remaining slots are filled using the order from Parent 2, skipping any cities that are already in the child's route (Paris, Rome, Berlin).

    • Parent 2's first city is Madrid. It's not in the child yet. Add it to the first empty spot.

      • [ Madrid, Paris, Rome, Berlin, _, _ ]

    • Next is Stockholm. Add it.

      • [ Madrid, Paris, Rome, Berlin, Stockholm, _ ]

    • Next is London. Add it.

      • [ Madrid, Paris, Rome, Berlin, Stockholm, London ]

    • The rest (Paris, Rome, Berlin) are skipped as they're already present from Parent 1.

    Illustration: Mutation

    Mutation introduces small, random changes into a child's chromosome after crossover. This is crucial for maintaining genetic diversity in the population and preventing the algorithm from getting stuck on a sub-optimal solution.

    Mutation Example: A Random Swap

    1. Take a child route that was just created.

    [ Madrid, Stockholm, Paris, Rome, Berlin, London ]

    [ Madrid, Berlin, Paris, Rome, Stockholm, London ]

    This simple swap has created a brand new route that didn't exist before, introducing a new possibility for the algorithm to explore.

    Real-World Applications

    Genetic algorithms are used to solve complex optimization problems across many industries where finding the "best" solution from a vast number of possibilities is critical.

    Engineering & Design

    GAs are used to create optimal designs for physical objects. For example, designing an antenna shape to maximize signal reception, an airplane wing for the best lift-to-drag ratio, or a lightweight bridge that can still support a heavy load. Each design parameter (length, angle, material) is a "gene."

    Logistics & Scheduling

    This is a classic application, very similar to our TSP simulation. GAs help companies optimize delivery routes to save fuel and time. They are also used for complex scheduling problems, like creating university timetables (assigning thousands of classes, professors, and rooms without conflicts) or scheduling nurses' shifts in a hospital.

    Financial Modeling

    In finance, GAs can be used to develop optimal trading strategies or build investment portfolios. A "chromosome" might represent a portfolio with different allocations to various stocks and bonds. The fitness function would aim to maximize returns while minimizing risk.

    Game Development & AI

    GAs can evolve intelligent behavior in non-player characters (NPCs) in video games. An AI opponent in a strategy game could "learn" better tactics by running thousands of simulated games and selecting the strategies (chromosomes) that lead to the most wins (highest fitness).

    Python Implementation

    Here is a simple, command-line implementation of a genetic algorithm to solve the Traveling Salesperson Problem. This code mirrors the logic used in the interactive simulation.

    import numpy as np
import random
import operator

# --- Create City and Fitness Classes ---
class City:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, city):
        xDis = abs(self.x - city.x)
        yDis = abs(self.y - city.y)
        distance = np.sqrt((xDis ** 2) + (yDis ** 2))
        return distance
    
    def __repr__(self):
        return "(" + str(self.x) + "," + str(self.y) + ")"

class Fitness:
    def __init__(self, route):
        self.route = route
        self.distance = 0
        self.fitness = 0.0
    
    def routeDistance(self):
        if self.distance ==0:
            pathDistance = 0
            for i in range(0, len(self.route)):
                fromCity = self.route[i]
                toCity = None
                if i + 1 < len(self.route):
                    toCity = self.route[i + 1]
                else:
                    toCity = self.route[0]
                pathDistance += fromCity.distance(toCity)
            self.distance = pathDistance
        return self.distance
    
    def routeFitness(self):
        if self.fitness == 0.0:
            self.fitness = 1 / float(self.routeDistance())
        return self.fitness

# --- GA Functions ---
def createRoute(cityList):
    route = random.sample(cityList, len(cityList))
    return route

def initialPopulation(popSize, cityList):
    population = []
    for i in range(0, popSize):
        population.append(createRoute(cityList))
    return population

def rankRoutes(population):
    fitnessResults = {}
    for i in range(0, len(population)):
        fitnessResults[i] = Fitness(population[i]).routeFitness()
    return sorted(fitnessResults.items(), key = operator.itemgetter(1), reverse = True)
