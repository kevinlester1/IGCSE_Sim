What is Association Rule Learning?

Association Rule Learning is an unsupervised machine learning method used to discover hidden patterns, correlations, or associations between variables in large databases. Unlike predictive modeling, its goal is not to predict a target variable, but rather to uncover rules that describe how items frequently group together.

Anatomy of a Rule: {A} → {B}

  • Antecedent (If): The item or itemset found in the data (e.g., Vandalism).
  • Consequent (Then): The item or itemset found in combination with the antecedent (e.g., Theft).

The 3 Core Evaluation Metrics

Support

Measures how popular an itemset is, as measured by the proportion of transactions in which an itemset appears.

Support(A) = Freq(A) / Total
Support(A→B) = Freq(A,B) / Total

Confidence

Measures how likely item B is purchased when item A is purchased. It represents the conditional probability of the rule.

Conf(A→B) = Support(A,B) / Support(A)

Lift

Measures the strength of an association over random chance. It accounts for the base popularity of the items.

Lift = Supp(A,B) / (Supp(A) * Supp(B))
Lift > 1
Positive correlation (A makes B more likely)
Lift = 1
Independent (No actual correlation)
Lift < 1
Negative correlation (A makes B less likely)

The Apriori Algorithm

Checking every possible combination of items in a large database takes too much time (exponential time complexity). The Apriori algorithm speeds this up using a simple pruning strategy.

The Apriori Principle (Anti-Monotonicity)

"If an itemset is infrequent, all of its supersets must also be infrequent."

Example: If {Milk, Bread} only appears 2 times (and fails the minimum support threshold), then {Milk, Bread, Eggs} cannot possibly appear more than 2 times. The algorithm instantly ignores evaluating {Milk, Bread, Eggs}, saving massive amounts of computation.

Real-World Applications

  • Predictive Policing (Criminology): Identifying geographical zones where specific combinations of crimes occur together to better deploy patrols.
  • Market Basket Analysis (Retail): Discovering that people who buy diapers on Friday evenings often buy beer, allowing stores to place them next to each other.
  • Healthcare: Mining patient records to find co-occurring symptoms, which can aid in earlier diagnosis of complex diseases.
  • Web Usage Mining: Understanding clickstreams. "If a user visits page A and page B, they are likely to click on link C."
