Hardware Explanations

  • ALU & CU: ALU performs math/logic; CU directs operations and data flow.
  • Registers:
    • PC: Holds address of next instruction.
    • MAR: Holds address to fetch/write from memory.
    • MDR: Holds actual data fetched/waiting to be written.
    • IR: Holds current instruction being decoded.
    • Accumulator: Stores intermediate arithmetic/logic results.
  • Buses:
    • Address Bus: Carries the memory location to be read/written (Unidirectional).
    • Data Bus: Carries actual data/instructions between CPU and memory (Bidirectional).
    • Control Bus: Carries command signals like Read, Write, and Clock synchronization.
  • Cores & Processors: A core is an independent processing unit in a CPU.
    • CPU: Few powerful cores optimized for sequential, general-purpose tasks.
    • GPU (Graphics Processing Unit): Thousands of simpler cores designed for highly parallel processing (ideal for graphics and AI/Machine Learning).
  • Instruction Pipelining (Deep Dive)

    Sequential vs. Pipelined: In a basic (sequential) CPU, an instruction must go completely through the Fetch, Decode, and Execute stages before the next instruction can even begin. This leaves parts of the CPU idle.

    How Pipelining Works: Instruction Pipelining overlaps the execution of multiple instructions to increase overall performance (throughput). It works like an assembly line. As soon as Instruction 1 finishes Fetching and moves to the Decode unit, the Fetch unit is immediately freed up to grab Instruction 2.

    Once the pipeline is "full," the CPU completes one instruction every single clock cycle, drastically speeding up execution.

  • Memory Types:
    • RAM (Random Access Memory): Volatile working memory storing currently active programs and data.
    • ROM (Read-Only Memory): Non-volatile memory storing essential bootup instructions (BIOS).
    • Cache: Small, ultra-fast memory near/on the CPU storing frequently used data. Divided into levels:
      • L1 Cache: Smallest and fastest, located directly on the CPU core.
      • L2 Cache: Larger but slightly slower, usually dedicated to a specific core.
      • L3 Cache: Largest and slowest cache, typically shared across all cores.

Storage & Cloud Explanations

  • Secondary Storage: Non-volatile, long-term storage.
    • Magnetic (HDD): High capacity, low cost, uses moving mechanical parts.
    • Solid State (SSD): Flash memory, no moving parts, faster, durable, higher cost.
    • Optical (CD/DVD): Lasers read/write data on a disc. Very portable, low capacity.
  • Data Compression: Reducing file sizes to save storage space and transmission time.
    • Lossy: Permanently removes non-essential data (e.g., JPEG, MP3).
    • Lossless: Reduces size without losing any data; perfectly restorable (e.g., ZIP).
    • Run-Length Encoding (RLE): A form of lossless compression that replaces sequences of identical data with a single count and the value (e.g., 'AAAA' becomes '4A').
  • Cloud Computing Models:
    • SaaS (Software as a Service): Ready-to-use software accessed via browser (e.g., Google Workspace).
    • PaaS (Platform as a Service): Provides development frameworks/tools for programmers without hardware management (e.g., Heroku).
    • IaaS (Infrastructure as a Service): Raw virtualized hardware (servers, storage). You manage the OS and apps (e.g., AWS EC2).
