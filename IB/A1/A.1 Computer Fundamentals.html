When are these used?

A

FCFS: Batch Jobs

Example: Printer Queues. A printer handles documents exactly as they arrive. If User A sends a 100-page book, User B's 1-page memo must wait. FCFS is "fair" in order but poor for high-priority small tasks.

B

RR: Modern Desktop OS

Example: Windows/macOS. Your music player (Spotify) needs constant CPU to avoid stuttering, while your Browser (Chrome) needs CPU for loading. RR gives each a 2ms "slice," ensuring both feel responsive at the same time.

C

Priority: Critical Systems

Example: Flight Control. In an airplane, a "Low Altitude Alert" task is far more important than a "Refresh In-Flight Entertainment" task. The critical safety task is assigned a higher priority and processed immediately.

How & When are they used?

Uses: Polling

1

Game Loops: Games poll the keyboard/mouse every single frame (e.g. 144Hz) to ensure movement is perfectly responsive without latency jitter.

2

High-Speed Sensors: In automation, a sensor might trigger 10,000+ times a second. The context-switching overhead for interrupts would be slower than just polling the buffer.

Uses: Interrupts

1

Keyboards: Humans type slowly (to a CPU). Interrupts allow the CPU to render graphics or run code until a key is actually pressed.

2

Network Cards: Data packets arrive at unpredictable times. The NIC card interrupts the CPU only when a full packet is ready in the buffer.

Resolving Deadlocks

1. Prevention

Modify the system to ensure one of the 4 Coffman conditions can never occur. e.g. Require process to request all resources at once.

2. Avoidance

OS denies unsafe requests that might lead to a deadlock cycle (e.g. Banker's Algorithm).

Recovery Methods

3. Detection & Recovery

Allow deadlocks, identify the cycle, and abort one of the processes or preempt resources to break the chain.

4. The Ostrich Algorithm

If extremely rare, ignore the deadlock entirely to save system overhead. System will require a manual reboot.

Control Systems Overview

Closed-Loop (Feedback)

The output of the system is continuously measured and fed back into the controller to adjust the input, maintaining a target state (e.g., HVAC). Adapts to external changes.

Open-Loop

Executes a fixed process based solely on the input, regardless of the actual output or result (e.g., a basic toaster timer). Cannot adapt to changes.

Hardware Stack

  • Sensors:Collects physical data (temp, light, motion).
  • Transducers:Converts physical energy into electrical signals (e.g., Analog-to-Digital Converter).
  • Algorithm:The logic loop ("If temp < target, heat").
  • Actuators:Mechanical outputs (motors, valves, heaters).

Translation Processes (HL)

A1.4.1 Evaluating Interpreters & Compilers

Compilers

Ahead-Of-Time (AOT)

Mechanics: Translates the entire source program into machine code (object code) to create a standalone executable file before execution begins.

Translation TimeSlow initially (translates everything at once), but execution is extremely fast.
Error DetectionFull list of syntax errors reported only after the entire compilation scan finishes.
PortabilityLow. The generated executable is tied to a specific hardware architecture/OS.
ApplicabilityFinal production builds, commercial software distribution.

Interpreters

Line-by-Line Runtime

Mechanics: Translates and executes instructions one line at a time directly from the source code during runtime. No executable is saved.

Translation TimeFast to start running, but overall execution is slower due to continuous translation overhead.
Error DetectionStops execution immediately upon encountering the first error, highlighting the exact line.
PortabilityHigh. Source code runs anywhere the interpreter software is installed.
ApplicabilityScripting, rapid prototyping, web development (JS, Python).

Modern Hybrid Methods

Bytecode Interpreters

Source code is pre-compiled into a highly optimized, platform-neutral intermediate code called Bytecode. A Virtual Machine (VM) then interprets this bytecode for the specific hardware (e.g., Java Virtual Machine). Combines syntax checking with high portability.

Just-In-Time (JIT) Compilation

Works alongside bytecode. Instead of interpreting bytecode line-by-line, the JIT compiler translates frequently used bytecode into native machine code during runtime. This provides near-compiled execution speed while maintaining cross-platform portability.

Vocabulary & Key Terms

Definitions for A1.3 to A1.4

Operating Systems & Scheduling

Virtual Memory

A memory management capability of an OS that uses hardware and software to allow a computer to compensate for physical memory shortages by temporarily transferring data from random access memory (RAM) to disk storage.

Device Driver

A software program that operates or controls a particular type of device that is attached to a computer.

Round Robin (RR)

A scheduling algorithm where each process is assigned a fixed time slot (quantum) in a cyclic order. It ensures fairness and prevents starvation.

First-Come, First-Served (FCFS)

A scheduling algorithm that automatically executes queued requests and processes by the order of their arrival.

Polling

The continuous checking of the status of a hardware device by the CPU to see if it needs attention (Busy-Waiting).

Interrupt

A signal to the processor emitted by hardware or software indicating an event that needs immediate attention.

HL Specific: Multitasking, Control & Translation

Deadlock

A situation in OS multitasking where two or more processes are unable to proceed because each is waiting for one of the others to do something.

Transducer

An electronic device that converts energy from one form to another. In control systems, it usually converts physical property measurements into electrical signals (e.g., ADC).

Actuator

A component of a machine that is responsible for moving and controlling a mechanism or system (e.g., a motor or valve).

Closed-Loop System

A control system that uses feedback from the output to control the input, allowing it to automatically correct errors.

Compiler

A translator that converts a high-level programming language into machine code in one single batch before execution.

Just-In-Time (JIT) Compilation

A method that translates intermediate code (bytecode) into machine code at runtime, combining the speed of compilation with the portability of interpretation.
