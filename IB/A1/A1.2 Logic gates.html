Simplification Mnemonics

The "Black Hole" Rules

Signals get "sucked in" and lost.

A · 0 = 0 | A + 1 = 1

The "Mirror" Rules

The signal just stays the same.

A · 1 = A | A + 0 = A

De Morgan's Secret

"Break the bar, change the sign."

¬(A · B) = ¬A + ¬B

Simplification Walkthrough

Example: Z = (A · B) + (A · B)

1. Factoring (Distributive Law)

Extract the common variable 'A' from both terms.

Z = A · (B + B)
2. Redundancy (Inverse Law)

Anything OR-ed with its opposite is always TRUE (1).

Z = A · (1)
3. Result (Identity Law)

Anything multiplied by 1 stays exactly as it is.

Z = A
Hardware Saving: We replaced 4 gates with a single wire!

Why Gray Code?

Notice the sequence 01 → 11 → 10. Only one bit changes at a time. This allows adjacent cells to be "factored out" logically because their physical proximity mirrors a mathematical relationship.

How to Construct a K-Map

Step 1: Size the Map

For 3 variables (A, B, C), you need 8 cells (2 raised to the power of 3). The grid is arranged so variables change one at a time.

Step 2: Transfer the 1s

Find every row in your Truth Table where the final output is 1. Put a "1" in the matching cell on the K-map grid.

Step 3: Group the 1s (The Rules)

  • Groups must contain 1, 2, 4, or 8 cells (powers of 2).
  • Groups must be as large as possible to maximize simplification.
  • Groups can wrap around from the left edge to the right edge.
  • Groups can overlap if it helps create a larger group.

Step 4: Reading the Result

Look at each group. Identify variables that stay the same (e.g., B is 1 in every cell of the group). Discard any variable that changes (e.g., A changes from 0 to 1).
