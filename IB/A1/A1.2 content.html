Hardware Reality: Why Binary?

Reliability

Hardware works with voltage. Detecting "High" vs "Low" is extremely reliable compared to trying to detect 10 different voltage levels accurately.

Signal Gap

Binary has a massive "safety gap". A 4.2V signal is still clearly a "1", even if it should have been 5V. This protects data from electrical interference.

Understanding Number Systems

Binary (Base-2)

Computers operate using billions of microscopic switches that can only be On (1) or Off (0). In binary, each position represents a power of 2 (1, 2, 4, 8, 16...). It is the fundamental, native language of all digital hardware.

Hexadecimal (Base-16)

Binary sequences get very long, making them hard for humans to read. Hexadecimal uses 16 symbols (0-9, and A-F for 10-15). Because exactly 4 binary bits (a nibble) perfectly equal 1 Hex digit, it acts as a highly efficient shorthand for binary.

Signed Integer Systems

1. Sign and Magnitude

The first bit is a flag: 0 (+), 1 (-). Flaw: Two zeros (+0, -0) and complex subtraction circuitry.

2. Two's Complement

CPU standard. The MSB has a negative weight (-128). Allows addition and subtraction to use the same logic.

Conversion: Positive bits → Flip all → Add 1.

Real Numbers: IEEE 754 Floating Point

To represent fractional numbers like 0.5 or very large numbers like 10³⁸ in binary, computers use Floating Point. This is essentially scientific notation: ±1.Mantissa × 2^Exponent.

1 Normalization

Computers "normalize" numbers to keep the binary point consistent. Exactly one non-zero digit stays on the left (e.g., 1.011).

The "Hidden Bit": Normalized binary always starts with "1", so we save space by not storing it.

2 The Biased Exponent

Uses a Bias of 127 to store negative powers safely without a second sign bit.

Formula: Stored Exponent = Actual Exponent + 127

Example A: 13.5₁₀

1. Binary: 1101.1₂

2. Normalize: 1.1011 × 2³

3. Exponent: 127 + 3 = 130 (10000010₂)

4. Mantissa: 1011... (Hidden 1)

Example B: 0.125₁₀

1. Binary: 0.001₂

2. Normalize: 1.0 × 2⁻³

3. Exponent: 127 - 3 = 124 (01111100₂)

Visual Data: Bitmaps & Compression

How Bitmaps are Formed

A Bitmap (Raster) image is formed by dividing an image into a grid of tiny colored squares called pixels. The number of pixels (Width × Height) is the Resolution. Each pixel's color is represented by a binary number.

BMP Binary Anatomy

FILE HEADER

Metadata identifier. Starts with bytes 'BM'. Stores total file size.

DIB HEADER

Image specifications: width/height and bit-depth.

PIXEL ARRAY

The actual payload. Raw list of binary RGB numbers scanned left-to-right, top-to-bottom.

Compression & File Types

.BMP

Uncompressed

Stores every pixel exactly as it is. Huge file sizes, perfect quality. Bad for web.

.PNG

Lossless Compression

Uses algorithms like Run-Length Encoding (RLE) to shrink file size without losing pixel data. Excellent for logos/text.

.JPG

Lossy Compression

Permanently discards visual data the human eye barely notices. Tiny file sizes, making it the standard for photos.

Analog to Digital: Audio

The 3-Step ADC Process

  • 1. Sampling: Taking continuous analog sound waves and capturing voltage "snapshots" at fixed intervals. Higher sampling rate = smoother digital wave.
  • 2. Quantization: Measuring each snapshot's wave height and rounding it to the nearest available digital value (determined by Bit Depth).
  • 3. Encoding: Storing those rounded measurement numbers as a sequence of binary bits.

    • Discrete Sample Points

    Nyquist Theorem

    To accurately record a sound, sample at twice its highest frequency. Humans hear up to 20kHz, so CDs sample at 44.1kHz.

    Bitrate & Quality

    Bitrate is the amount of data processed per second.
    Bitrate = Sample Rate × Bit Depth × Channels

    Calculating Audio File Size

    Size (Bytes) = (Sample Rate × Bit Depth × Channels × Seconds) ÷ 8

    Video Compression & Formats

    Compression Approaches

    Raw video takes up massive amounts of storage (e.g., 10 seconds of uncompressed 1080p is ~1.5 GB). To make streaming possible, compression is heavily applied:

    Spatial (Intra-frame) Compression

    Compresses each frame individually, similar to JPEG image compression. It removes redundant pixel data within the same picture.

    Temporal (Inter-frame) Compression

    Instead of storing every frame completely, it only saves the differences (deltas) between frames. Static backgrounds are kept in memory and reused.

    Codecs (The Compressors)

    H.264 (AVC): The most widely used codec globally. Highly compatible with all web devices.
    H.265 (HEVC): 50% more efficient than H.264. The standard for 4K and high-end video.
    AV1 / VP9: Modern, royalty-free codecs heavily used by platforms like YouTube and Netflix.

    Containers (The File Types)

    .MP4: The universal container. Can package video (usually H.264), audio, and subtitles together.
    .MKV: An open-source container that supports unlimited audio and subtitle tracks.
    .MOV / .AVI: Legacy containers created by Apple and Microsoft, respectively.

    Note: A file extension like .MP4 is just a "box" or wrapper. The actual binary data inside is compressed by a codec. You can have one MP4 file that uses H.264, and another MP4 file that uses H.265.
