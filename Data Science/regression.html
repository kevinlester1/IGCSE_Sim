Beyond Two Dimensions

So far, we've only used one variable (house size) to predict the price. This is called univariate regression.

In the real world, many factors affect a house's price. We could include variables like the number of bedrooms, age of the house, or distance to the city center. When we use multiple variables, it's called multivariate regression.

The Formula Expands

Our simple line equation was:

\( \text{Price} = b_0 + m_1 \cdot \text{Size} \)

For a multivariate model, we just add more terms—one for each new variable. Each new variable gets its own gradient (or coefficient).

\( \text{Price} = b_0 + m_1 \cdot \text{Size} + m_2 \cdot \text{Bedrooms} + m_3 \cdot \text{Age} \)

  • \(b_0\) is still the intercept.
  • \(m_1\) is the gradient for Size.
  • \(m_2\) is the gradient for Bedrooms.
  • \(m_3\) is the gradient for Age.

From a Line to a Plane

With one predictor variable (size), we find the best-fitting line. If we add a second predictor (e.g., number of bedrooms), our data points now exist in 3D space. The model's goal is to find the best-fitting plane that slices through the data points, minimizing the squared errors just like before.

Instead of a 2D line, a model with two variables creates a 3D plane.

If we add a third variable, we can't easily visualize it anymore! The model creates a "hyperplane" in 4D space. The math is the same, but our ability to picture it breaks down.

Does the Model Generalize?

A model that perfectly describes the data it was trained on is useless if it can't make accurate predictions on new, unseen data. This is why testing is critical.

The Train-Test Split

The standard method is to split your dataset into two parts:

  • Training Set (e.g., 80% of data): The model learns the relationships from this data. The intercept and gradient(s) are calculated using only these points.
  • Testing Set (e.g., 20% of data): This data is held back. The model never sees it during training. We use it at the very end to evaluate performance.

The core principle of splitting data to test a model's real-world performance.

Common Evaluation Metrics

After training, we feed the test set's input variables (e.g., house sizes) to the model and compare its predictions to the actual prices. Here's how we measure the error:

Mean Absolute Error (MAE)

The average of the absolute differences between predicted and actual values. It's easy to interpret: "On average, our model's prediction is off by $X."

\( \text{MAE} = \frac{1}{n} \sum_{i=1}^{n} | \text{Actual}_i - \text{Predicted}_i | \)

Root Mean Squared Error (RMSE)

This is the square root of the average of the squared errors. It penalizes large errors more heavily than MAE and is in the same unit as the output (e.g., dollars).

\( \text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (\text{Actual}_i - \text{Predicted}_i)^2} \)

A good model will have low MAE and RMSE values on the test set. A large difference between training error and testing error often indicates overfitting.

How to Interpret This Chart

  • Each point's vertical position shows how far the model's prediction was from the actual price for that house.
  • The Goal: We want to see a random, "shotgun blast" pattern of points scattered around the zero line.
  • Warning Signs: If you see a clear pattern (like a U-shape, a funnel, or points consistently on one side), it means a simple straight line isn't capturing the full story in the data.
