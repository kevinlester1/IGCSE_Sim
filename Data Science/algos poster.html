Search Algorithm

Binary Search

O(log n)

How It Works

  1. Identify Bounds: Start with two pointers, one at the beginning (`low`) and one at the end (`high`) of the sorted list.
  2. Calculate Midpoint: Find the middle index: `mid = (low + high) / 2`.
  3. Compare: Check the value at the middle index.
    • If it matches the target, you are done.
    • If the target is smaller, discard the right half (`high = mid - 1`).
    • If the target is larger, discard the left half (`low = mid + 1`).
  4. Repeat: Continue this process with the new, smaller range until the target is found or the range is empty.
FUNCTION BinarySearch(arr, target): low = 0 high = length(arr) - 1 WHILE low <= high: mid = floor((low + high) / 2) IF arr[mid] == target: RETURN mid // Target found ELSE IF arr[mid] < target: low = mid + 1 // Target is in right half ELSE: high = mid - 1 // Target is in left half RETURN -1 // Target not found
Real World Use Cases Used in database indexing (SQL), debugging (git bisect), dictionary lookups, and standard library sorting functions like `std::lower_bound`.
Sorting Algorithm

Quick Sort

O(n log n)

How It Works

  1. Pick a Pivot: Select an element from the array to serve as the "pivot" (can be first, last, random, or median).
  2. Partition: Reorder the array so that all elements with values less than the pivot come before it, and all elements greater than the pivot come after it. The pivot is now in its final sorted position.
  3. Recursion: Recursively apply the same logic to the sub-array of smaller elements and the sub-array of larger elements.
  4. Base Case: Recursion stops when sub-arrays have 0 or 1 element, as they are inherently sorted.
FUNCTION QuickSort(arr, low, high): IF low < high: // Partition the array and get pivot index pi = Partition(arr, low, high) // Recursively sort elements before and after partition QuickSort(arr, low, pi - 1) QuickSort(arr, pi + 1, high) FUNCTION Partition(arr, low, high): pivot = arr[high] i = low - 1 FOR j = low TO high - 1: IF arr[j] < pivot: i++ Swap arr[i] with arr[j] Swap arr[i + 1] with arr[high] RETURN i + 1
Real World Use Cases Default sorting algorithm in many languages (C++ `std::sort`, Java `Arrays.sort` for primitives) due to cache efficiency and in-place sorting capabilities.
Sorting Algorithm

Merge Sort

O(n log n)

How It Works

  1. Divide: Find the middle index of the array and split it into two halves.
  2. Conquer: Recursively call Merge Sort on the first half and the second half. Continue until sub-arrays contain only 1 element (which is always sorted).
  3. Merge: Compare the smallest elements of the two sub-arrays.
    • Place the smaller of the two into a new result array.
    • Move the pointer of that sub-array forward.
    • Repeat until one array is empty, then append the remaining elements.
FUNCTION MergeSort(arr): IF length(arr) <= 1: RETURN arr mid = length(arr) / 2 left = MergeSort(arr[0...mid]) right = MergeSort(arr[mid...end]) RETURN Merge(left, right) FUNCTION Merge(left, right): result = [] WHILE left is not empty AND right is not empty: IF left[0] <= right[0]: Append left[0] to result ELSE: Append right[0] to result Append remaining items to result RETURN result
Real World Use Cases Preferred for sorting Linked Lists (random access not needed), E-commerce product sorting (stable sort preserves previous order), and handling massive datasets on disk (external sorting).
Graph Algorithm

Dijkstra's

O(V + E log V)

How It Works

  1. Initialization: Set distance to the start node to 0 and all other nodes to infinity. Mark all nodes as unvisited.
  2. Selection: From the unvisited nodes, select the one with the smallest known distance (use a Priority Queue for efficiency).
  3. Relaxation: Check all neighbors of the selected node.
    • Calculate new_distance = current_node_dist + edge_weight.
    • If new_distance < neighbor's old distance, update it.
  4. Completion: Mark the current node as visited. Repeat until the destination is reached or all reachable nodes are visited.
FUNCTION Dijkstra(Graph, start_node): distances = {all_nodes: infinity} distances[start_node] = 0 priority_queue.add(start_node, 0) WHILE priority_queue is not empty: current_node = priority_queue.pop_min() FOR neighbor, weight IN current_node.edges: new_dist = distances[current_node] + weight IF new_dist < distances[neighbor]: distances[neighbor] = new_dist priority_queue.update(neighbor, new_dist) RETURN distances
Real World Use Cases GPS Navigation systems (Google Maps, Waze) to find shortest routes, IP routing protocols (OSPF), and social network connections.
Data Structure

Hash Maps

O(1)

How It Works

  1. Key Input: You provide a "Key" (e.g., a username, product ID).
  2. Hashing: The key is passed through a Hash Function. This function performs mathematical operations to convert the key into a unique integer (hash code).
  3. Mapping: The integer is mapped (usually via Modulo `%`) to a valid index in the storage array.
  4. Storage/Collision: The value is stored at that index. If two keys result in the same index (Collision), they are stored in a list at that index (Chaining) or the algorithm looks for the next empty slot (Open Addressing).
CLASS HashMap: array_size = 16 buckets = [[] for _ in range(array_size)] FUNCTION put(key, value): index = hash(key) % array_size // Check if key exists and update for pair in buckets[index]: if pair.key == key: pair.value = value return // Else append new pair buckets[index].append(Pair(key, value)) FUNCTION get(key): index = hash(key) % array_size for pair in buckets[index]: if pair.key == key: return pair.value return null
Real World Use Cases Caching (Redis), Database Indexing, Symbol Tables in Compilers, Counting frequencies of items, and detecting duplicates.
Graph Traversal

BFS vs DFS

O(V + E)

How It Works

BFS (Breadth-First Search)
  1. Use a Queue (FIFO).
  2. Add start node to queue.
  3. Remove node, visit it, add all unvisited neighbors to queue.
  4. Result: Ripples out layer by layer.
DFS (Depth-First Search)
  1. Use a Stack (LIFO) or Recursion.
  2. Visit node, pick one unvisited neighbor.
  3. Repeat until stuck, then backtrack.
  4. Result: Drills down deep paths first.
FUNCTION BFS(start_node): queue = [start_node] visited = {start_node} WHILE queue not empty: current = queue.dequeue() FOR neighbor in current.adj: IF neighbor not in visited: visited.add(neighbor) queue.enqueue(neighbor) FUNCTION DFS_Recursive(node, visited): visited.add(node) FOR neighbor in node.adj: IF neighbor not in visited: DFS_Recursive(neighbor, visited)
Real World Use Cases BFS: Shortest path in unweighted graphs, Social Media (people within 2 degrees of connection).
DFS: Solving mazes, Cycle detection, Topological Sorting, and Game AI analysis.
Optimization

Dynamic Programming

Varies

How It Works

  1. Identify Recursive Structure: The problem must be breakable into overlapping sub-problems (e.g., Fibonacci(n) = Fib(n-1) + Fib(n-2)).
  2. Check Cache (Memoization): Before computing a sub-problem, check if the result is already stored in a table/dictionary.
  3. Compute & Store: If not cached, solve the sub-problem and store the result for future use.
  4. Construct Solution: Use the stored results of sub-problems to build the answer for the main problem.
// Top-Down with Memoization (Fibonacci) memo = {} FUNCTION Fib(n): IF n <= 1: RETURN n // 1. Check Cache IF n in memo: RETURN memo[n] // 2. Compute result = Fib(n-1) + Fib(n-2) // 3. Store memo[n] = result RETURN result
Real World Use Cases Diff utilities (Longest Common Subsequence), Text justification, Finance (Knapsack problem for resource allocation), and Seam Carving (image resizing).
Cryptography

RSA Encryption

O(k⁴)

How It Works

  1. Key Generation: Select two massive prime numbers, `p` and `q`. Calculate `n = p * q`. The security relies on the fact that factoring `n` back into `p` and `q` is computationally impossible for large numbers.
  2. Public Key (e): Choose a public exponent `e`. The pair `(e, n)` is shared publicly.
  3. Private Key (d): Compute `d` such that it is the modular inverse of `e`. This `d` is kept secret.
  4. Encrypt/Decrypt:
    • Encrypt: `Ciphertext = Message^e mod n`
    • Decrypt: `Message = Ciphertext^d mod n`
FUNCTION KeyGen(): p = LargePrime(); q = LargePrime() n = p * q phi = (p-1) * (q-1) e = 65537 // Common choice d = ModularInverse(e, phi) RETURN (pk=(e,n), sk=(d,n)) FUNCTION Encrypt(msg, pk): RETURN power(msg, pk.e) % pk.n FUNCTION Decrypt(cipher, sk): RETURN power(cipher, sk.d) % sk.n
Real World Use Cases HTTPS/SSL Certificates (securing the web), SSH keys, Digital Signatures, and secure email (PGP).
Machine Learning

K-Means

NP-Hard

How It Works

  1. Initialize: Choose `K` random points to act as the initial center points (centroids) for your clusters.
  2. Assignment: For every data point, calculate the distance to each centroid. Assign the point to the cluster of the closest centroid.
  3. Update: Calculate the average position (mean) of all points inside each cluster. Move the centroid to this new average position.
  4. Repeat: Repeat steps 2 and 3 until the centroids stop moving (convergence).
FUNCTION KMeans(data, k): centroids = RandomPoints(k) WHILE not converged: clusters = [[] for _ in range(k)] // 1. Assign points FOR point IN data: closest = FindNearest(point, centroids) clusters[closest].append(point) // 2. Re-center old_centroids = centroids FOR i FROM 0 TO k: centroids[i] = Average(clusters[i]) IF centroids == old_centroids: BREAK RETURN clusters
Real World Use Cases Customer segmentation (grouping shoppers by behavior), Image compression (color quantization), Anomaly detection, and Document clustering.
Network Analysis

PageRank

O(n+m)

How It Works

  1. Vote by Link: Every link from Page X to Page Y is considered a "vote" for Page Y.
  2. Weighted Votes: Not all votes are equal. A vote from a high-ranking page counts far more than a vote from a low-ranking page.
  3. Distribution: A page splits its own rank evenly among all the pages it links to.
  4. Damping Factor: To prevent loops (traps), there is a small probability (usually 0.15) that a user jumps to a random page instead of following a link.
FUNCTION PageRank(graph, iterations): ranks = {node: 1/N for node in graph} d = 0.85 // Damping factor FOR i FROM 0 TO iterations: new_ranks = {} FOR node IN graph: incoming_score = 0 FOR linker IN node.inbound_links: incoming_score += ranks[linker] / NumOutLinks(linker) new_ranks[node] = (1 - d) + d * incoming_score ranks = new_ranks RETURN ranks
Real World Use Cases Google Search (originally), Recommendation Engines (Twitter's "Who to Follow"), Biology (protein interaction networks), and Citation Analysis.
