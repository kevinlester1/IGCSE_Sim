Part 1: CPU Architecture & Performance

Hardware Explanations

  • ALU & CU: ALU performs math/logic; CU directs operations and data flow.
  • Registers:
    • PC (Program Counter): Holds address of next instruction.
    • MAR (Memory Address Register): Holds address to fetch/write from memory.
    • MDR (Memory Data Register): Holds actual data fetched/waiting to be written.
    • CIR (Current Instruction Register): Holds the current instruction being decoded and executed.
    • Accumulator (ACC): Stores intermediate arithmetic/logic results.
  • Buses:
    • Address Bus: Carries the memory location to be read/written (Unidirectional).
    • Data Bus: Carries actual data/instructions between CPU and memory (Bidirectional).
    • Control Bus: Carries command signals like Read/Write.
  • CPU Performance Factors

    Clock Speed: Measured in Hertz (Hz). 1 GHz = 1 billion FDE cycles per second. A faster clock means more instructions processed per second.

    Number of Cores: A dual-core or quad-core processor can theoretically execute multiple instructions simultaneously, multiplying performance (if software supports it).

    Cache Size: Extremely fast memory located directly on the CPU. Larger cache means the CPU rarely has to wait for the slower RAM, speeding up processing.

  • Embedded Systems:

    A computer system built within a larger device, designed for a specific control function. Examples include washing machines, microwaves, car engine management systems, and vending machines. They often have programs stored permanently in ROM.

CPU Vocabulary

Accumulator (ACC)
A register that temporarily stores results of ALU operations.
Microprocessor
An integrated circuit on a single chip that functions as a CPU.
Embedded System
A dedicated computer system built into a larger device to control it.
CIR
Current Instruction Register - holds the instruction currently being decoded.

Part 2: Memory & Storage

Memory & Storage Explanations

  • Primary Memory: Directly accessed by the CPU.
    • RAM (Random Access Memory): Volatile memory storing currently running programs and data.
    • ROM (Read Only Memory): Non-volatile memory storing the bootstrap loader/start-up instructions.
  • Virtual Memory:

    When RAM fills up, the OS uses part of the secondary storage (hard drive) to act as temporary RAM. It prevents errors but is much slower due to "disk thrashing" as data swaps back and forth.

  • Secondary Storage: Non-volatile, long-term storage.
    • Magnetic (HDD/Tape): High capacity, low cost, uses platters and moving read/write heads. Susceptible to physical damage.
    • Solid State (SSD/Flash): Uses NAND/NOR flash memory gates. No moving parts, faster, durable, but higher cost per GB.
    • Optical (CD/DVD/Blu-ray): Lasers read/write data by reflecting off 'pits' and 'lands' on a disc. Portable but low capacity.
  • Cloud Storage: Saving data on remote servers accessed via the internet.
    • Advantages: 24/7 availability, built-in redundancy/backups, easy synchronisation across devices, scalable storage.
    • Disadvantages: Requires internet connection, slower transfer speeds for large files, potential privacy/security concerns relying on a third party.

Storage Vocabulary

Volatile
Data is lost when the power is turned off (e.g., RAM).
Non-Volatile
Data is retained even without power (e.g., ROM, HDD).
Pits and Lands
The physical bumps and flat areas on an optical disc used to encode binary data.
