Register Reference Guide

PC

Program Counter

Holds the address of the NEXT instruction to be executed. Increments by 1 immediately after the fetch.

MAR

Memory Address Register

Holds the memory address of the CURRENT instruction, or the data it uses, so it can be fetched from RAM.

MDR

Memory Data Register

Holds the actual instruction or data that has just been fetched from RAM (or is waiting to be written to RAM).

CIR

Current Instruction Register

Holds the instruction CURRENTLY being decoded and executed by the Control Unit.

ACC

Accumulator

Holds the result of an arithmetic or logic instruction temporarily before it is transferred to memory.

The 3 Buses

Address Bus (Uni-directional): Carries the "Where" information. Transmits memory addresses from the CPU (MAR) to RAM.

Data Bus (Bi-directional): Carries the "What" information. Moves actual data and instructions back and forth between CPU and RAM.

Control Bus (Uni-directional): Carries the "Action" signal. Sends commands like "READ" or "WRITE" from the CU to hardware components.

CPU Performance Factors

Clock Speed: Measured in Hertz (Hz). Higher speed = more instructions processed per second.

Number of Cores: Duplicate processing units (dual-core, quad-core) allow simultaneous task execution.

Cache Size: Fast memory on the CPU. Stores frequently used data to avoid slow fetches from RAM.
