The Program Development Life Cycle

This is a structured process used by a developer to build high-quality software. The "Waterfall" model has four key stages, completed in order.

The Four Stages

This stage is about understanding the problem. Before writing any code, you must be clear on what the software needs to do.

  • Problem Identification: Clearly define the problem the software will solve.
  • Identify Requirements: Create a specific list of what the software must do.
  • Abstraction: Remove unnecessary details to focus on the essential parts of the problem.
  • Decomposition: Break down the large problem into smaller, more manageable parts.

In this stage, you plan *how* the software will meet the requirements. This is the blueprint for the program.

  • Structure Diagrams: Visually break down the system into its component parts.
  • Flowcharts: Create diagrams to show the flow and logic of algorithms.
  • Pseudocode: Write out the logic of the program in a human-readable format.
  • Decomposition: Break down the problem into inputs, processes, outputs, and storage.

This is the stage where the program is actually written using a programming language, based on the plans from the Design stage.

  • Writing Program Code: Translating the design into a functional program.
  • Iterative Testing: Programmers test their code as they write it.

Once the code is complete, it is formally tested to ensure it works as expected and meets all the requirements.

  • Create a Test Plan: A plan is created with specific test data and the expected outcomes.
  • Execute Tests: The program is run with the test data.
  • Compare Results: The actual output is compared to the expected output.
  • Fix and Re-test: Any bugs are fixed, and the software is tested again.

Example: School Canteen Ordering App

1. Analysis

The problem is that canteen queues are too long. We need an app to let students pre-order food.

  • Requirements: View menu, add items to basket, pay. Canteen needs to see orders.
  • Abstraction: We only need the student's name and their order.
  • Decomposition: Break into: User Login, Menu Display, Order Processing, Payment.

2. Design

We plan the app's structure and logic. We create a blueprint for the coders.

  • UI Sketch: Draw a simple layout for the menu and checkout pages.
  • Flowchart: Diagram the steps from logging in to confirming an order.
  • Pseudocode: `totalCost <- 0; FOR EACH item in basket; totalCost <- totalCost + item.price; END FOR`.

3. Coding

Developers use a language like Python or JavaScript to build the app from the design.

  • Build the login screen.
  • Create the page that shows the menu items.
  • Write the shopping basket code, translating the pseudocode.

4. Testing

We check if the app works correctly and find any bugs.

  • Test Plan: Test adding 3 items. Expected result: Total cost is correct.
  • Normal Data: Test with typical orders (e.g., 2 items).
  • Extreme Data: Test with an empty basket and a very large order.
  • Erroneous Data: Test ordering 1.5 sandwiches. The app shouldn't crash.

Algorithms & Pseudocode

An algorithm is a step-by-step set of instructions to solve a problem. Pseudocode is a way to write algorithms using structured English, focusing on logic without worrying about the specific syntax of a programming language.

Building Blocks

  • INTEGER: A whole number. 10, -5, 0
  • REAL: A number with a decimal point. 19.99, -0.5
  • BOOLEAN: A value that can only be TRUE or FALSE.
  • CHAR: A single character. 'A', '%', '7'
  • STRING: A sequence of characters (text). "Hello World"
Symbol/KeywordMeaning
<-Assignment (e.g., score <- 0)
INPUTGet input from the user
OUTPUTDisplay information to the user
=, <>, >, <, >=, <=Comparison Operators
IF...THEN...ELSE...ENDIFSelection statement
FOR, WHILE, REPEAT UNTILIteration (looping) statements

Control Structures

1. Sequence

Instructions are executed one after another, in the order they are written.

OUTPUT "Enter price:"
INPUT itemPrice
total <- itemPrice * 1.2

2. Selection (IF statements)

A choice is made between different paths of execution based on a condition (TRUE/FALSE).

INPUT age
IF age >= 18 THEN
  OUTPUT "Adult"
ELSE
  OUTPUT "Child"
ENDIF

3. Iteration (Loops)

A block of code is repeated a certain number of times or until a condition is met.

FOR Loop (Count-controlled):

FOR i <- 1 TO 5
  OUTPUT "Hello"
NEXT i
// Outputs "Hello" 5 times

WHILE Loop (Condition-controlled):

count <- 0
WHILE count < 3 DO
  OUTPUT "Looping"
  count <- count + 1
ENDWHILE
