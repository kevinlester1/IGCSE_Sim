Creating Maintainable Programs

An Interactive Simulation

Your Mission:

You've inherited some code to calculate a triangle's area. It works, but it's hard to understand. Your task is to refactor it step-by-step to make it clean, readable, and easy to maintain.

Step 1: The Original Code

Here's the code your colleague left you. What makes this code hard to read, edit, and understand?

FUNCTION triA(a, b) c <- 0.5 * a * b RETURN c ENDFUNCTION OUTPUT triA(5, 3)

Think about the function name, variables, comments, and structure. Let's fix it together.

Improvement 1: Use Meaningful Identifiers

Code should be self-documenting. Using clear names for variables and functions is the most important step towards maintainability.

Before

FUNCTION triA(a, b) c <- 0.5 * a * b RETURN c ENDFUNCTION
  • What is `triA`?
  • What do `a` and `b` represent?
  • What is `c`?

After

FUNCTION triangleArea(base, height) area <- 0.5 * base * height RETURN area ENDFUNCTION

Much better! The purpose is now clear just from reading the code.

Improvement 2: Add Relevant Comments

Good comments explain the "why," not the "what." They clarify the purpose of a block of code or a complex line.

Avoid commenting on obvious things. For example, `RETURN area` doesn't need a comment saying "This returns the area." Instead, comment on the function's overall goal or the formula being used.

// This function calculates the area of a triangle given its base and height. FUNCTION triangleArea(base, height) area <- 0.5 * base * height // Formula: 1/2 * base * height RETURN area ENDFUNCTION

Improvement 3: Use Proper Indentation

Indentation makes code structure immediately visible. It shows which lines of code belong inside a function, loop, or condition.

Without Indentation

FUNCTION triangleArea(base, height) area <- 0.5 * base * height RETURN area ENDFUNCTION

Hard to see the function's body at a glance.

With Indentation

FUNCTION triangleArea(base, height) area <- 0.5 * base * height RETURN area ENDFUNCTION

The structure is clear. Some languages, like Python, require it!

Improvement 4: Use Functions Effectively

Functions make code modular and reusable. Our original code already used a function, which is great! The key is to make them flexible by using parameters.

Poor: Non-Reusable

FUNCTION triangleAreaFiveByThree() area <- 0.5 * 5 * 3 RETURN area ENDFUNCTION

This only works for one specific triangle. It's not maintainable.

Good: Reusable

FUNCTION triangleArea(base, height) area <- 0.5 * base * height RETURN area ENDFUNCTION

This works for any triangle by passing in arguments. It's flexible and reusable.

Mission Complete! ✅

By applying these principles, you've transformed the original code into a program that is easy to read, understand, and modify.

Final Maintainable Code:

// This function calculates the area of a triangle given its base and height. FUNCTION triangleArea(base, height) area <- 0.5 * base * height // Formula: 1/2 * base * height RETURN area ENDFUNCTION -- Example Usage -- OUTPUT triangleArea(5, 3)

Key Takeaways:

  • Use meaningful identifiers.
  • Write relevant comments to explain the 'why'.
  • Use indentation to show code structure.
  • Create reusable functions with parameters.
