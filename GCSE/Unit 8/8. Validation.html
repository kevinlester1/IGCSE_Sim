Interactive Data Validation Simulations

Enter data into the fields below to see different validation checks in action.

1. Presence Check

Ensures the input is not empty. Scenario: A username is required.

PROCEDURE presence_check()
  INPUT username
  IF username = "" THEN
    OUTPUT "Error: Cannot be empty."
  ELSE
    OUTPUT "Success."
  END IF
END PROCEDURE

2. Length Check

Verifies the input is at least a certain number of characters. Scenario: Password must be 8+ characters.

PROCEDURE length_check()
  INPUT password
  IF LENGTH(password) < 8 THEN
    OUTPUT "Error: Too short."
  ELSE
    OUTPUT "Success."
  END IF
END PROCEDURE

3. Range Check

Ensures a number is within a specific range. Scenario: Age must be between 18 and 99.

PROCEDURE range_check()
  INPUT age
  IF age < 18 OR age > 99 THEN
    OUTPUT "Error: Out of range."
  ELSE
    OUTPUT "Success."
  END IF
END PROCEDURE

4. Type Check

Confirms the data is of the correct type. Scenario: Quantity must be a whole number (integer).

PROCEDURE type_check()
  INPUT quantity
  IF IS_INTEGER(quantity) = FALSE THEN
    OUTPUT "Error: Not an integer."
  ELSE
    OUTPUT "Success."
  END IF
END PROCEDURE

5. Format Check

Checks if data follows a specific pattern. Scenario: Product code must be a letter then three digits (e.g., A123).

PROCEDURE format_check()
  INPUT productCode
  // Regex: Letter then 3 digits
  IF MATCHES_FORMAT(productCode) THEN
    OUTPUT "Success."
  ELSE
    OUTPUT "Error: Invalid format."
  END IF
END PROCEDURE

6. Check Digit

Validates a number using a calculated final digit. Scenario: '4837' -> sum is 22 -> check digit is 2. Full number: '48372'.

PROCEDURE check_digit()
  INPUT number
  // Separate base & last digit
  base ← SUBSTRING(number, 1, 4)
  provided ← SUBSTRING(number, 5, 1)

  // Calculate correct digit
  sum ← 0
  FOR each digit IN base
    sum ← sum + digit
  END FOR
  calculated ← sum MOD 10
  
  IF calculated = provided THEN
    OUTPUT "Success."
  ELSE
    OUTPUT "Error: Invalid check digit."
  END IF
END PROCEDURE
