Types of Test Data

When we test a program, we use different types of data to make sure it works correctly and can handle unexpected inputs. Here are the main types:

  • Normal Data: This is the type of data that you would expect a user to enter. It's valid, sensible, and falls well within the accepted range.
  • Extreme Data: This is data that is still valid but is at the very limits of the acceptable range. For example, the smallest and largest possible numbers.
  • Boundary Data: This is similar to extreme data, but it checks the values on both sides of the boundary. You test the last valid value and the first invalid value to see if the program handles the "edge cases" correctly.
  • Abnormal Data: This is data that should be rejected by the program. It can be of the wrong data type (e.g., text instead of a number) or completely outside the valid range.
